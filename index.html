One coherent stack

From satellite fix to browser map

01

Tracker

Acquires GNSS fixes, adapts its sleep policy, compresses route history, and retries until a gateway acknowledges delivery.

02

Gateway

Receives multiple trackers, validates frames, suppresses duplicates, and publishes stable point events to MQTT.

03

Archiver

Stores observations in SQLite, combines receptions from multiple gateways, and serves chunked history requests.

04

Web app

Connects over MQTT WebSockets, keeps a local IndexedDB cache, and renders recent routes without a server framework.

Designed for evolution

Stable boundaries, replaceable parts

Offline by default

RTC-backed queues and acknowledged delivery tolerate weak radio coverage and gateway outages.

Strict contracts

Transport, history, configuration, onboarding, and MQTT payloads reject unsupported schemas.

Multi-gateway reception

A stable point identity lets several gateways hear the same packet without duplicating history.

Inspectable operation

Plain C++, Python, SQLite, MQTT, and browser JavaScript keep the full path understandable.
