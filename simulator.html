Simulator

Repeatable tests without radio hardware

The brokerless suite reuses production validation, archival, service and embedded contract code instead of maintaining a separate toy model.

Run the complete suite

cd components/archiver
python -m lora_tracker_archiver.simulator \
  --trackers 2 \
  --points 12 \
  --service-suite \
  --embedded-suite

The result is a JSON summary. The output database is reset for deterministic runs unless you supply an explicit path.

What it covers

✓ Strict current schemas
✓ Multi-tracker routes
✓ Stable point identities
✓ Duplicate receptions
✓ Untimed-point fallback
✓ SQLite durability path
✓ History pagination
✓ MQTT service callbacks
✓ Allowlist rejection
✓ Both embedded header copies

What PlatformIO adds

The repository also compiles and links the V2/SX1276 tracker, Wireless Tracker ESP32-S3/SX1262 tracker and V2/SX1276 gateway. A real build caught an RTC memory overflow that native header tests could not see. Run both layers.

What it cannot prove

The simulator cannot validate RF range or airtime compliance, GNSS acquisition, deep-sleep current, battery aging, NVS/RTC recovery, real broker persistence, browser quotas, enclosure effects or button wiring. Those require hardware-in-the-loop, broker/browser end-to-end tests and environmental qualification.

Important: passing simulation is necessary contract evidence, not a production approval.
