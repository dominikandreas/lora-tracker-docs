Simulator

Repeatable tests without radio hardware

The brokerless suite reuses production validation, archival, service and embedded contract code instead of maintaining a separate toy model.

Run the complete suite

cd components/archiver
python -m lora_tracker_archiver.simulator \
  --trackers 2 \
  --points 12 \
  --service-suite \
  --embedded-suite

The result is a JSON summary. The output database is reset for deterministic runs unless you supply an explicit path.

What it covers

✓ Strict current schemas
✓ Multi-tracker routes
✓ Stable point identities
✓ Duplicate receptions
✓ Untimed-point fallback
✓ SQLite durability path
✓ History pagination
✓ MQTT service callbacks
✓ Relay hop bounds
✓ Relay suppression timing
✓ ACK cache lifetime
✓ All three header copies

What PlatformIO adds

The repository compiles five targets: two tracker boards, one gateway and the repeater role on both supported boards. Real target builds catch board-library and memory-layout failures that native contracts cannot see. Run both layers.

What it cannot prove

The simulator cannot validate RF range, hidden-node/capture effects, real multi-hop timing or airtime compliance, GNSS acquisition, deep-sleep current, battery aging, NVS/RTC recovery, real broker persistence, browser quotas, enclosure effects or button wiring. Those require hardware-in-the-loop, broker/browser end-to-end tests and environmental qualification.

Important: passing simulation is necessary contract evidence, not a production approval.
