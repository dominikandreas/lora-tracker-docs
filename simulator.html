Simulator

Repeatable tests without radio hardware

The simulator deliberately reuses production validation, archival, service, and embedded contract code.

Run the complete suite

cd components/archiver
python -m equine_archiver.simulator \
  --trackers 2 \
  --points 12 \
  --service-suite \
  --embedded-suite

The result is a JSON summary. A temporary SQLite database is reset on each run unless an explicit output path is supplied.

What it covers

✓ Multi-tracker routes
✓ Stable point identities
✓ Two gateway receptions
✓ Duplicate suppression
✓ Payload validation
✓ Timestamp fallback
✓ SQLite history
✓ Pagination and chunks
✓ MQTT service callbacks
✓ Shared firmware headers

What still needs production testing

The host suite cannot validate ESP32 peripherals, GNSS acquisition, deep sleep, flash recovery, real LoRa range and interference, broker TLS and reconnect behavior, or browser storage limits. Use PlatformIO builds, hardware-in-the-loop tests, a real broker, and browser end-to-end checks before field deployment.

Important: the embedded suite compiles shared protocol and configuration headers for both firmware components. It is not an ESP32 sketch build.
