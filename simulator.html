Explainable, deterministic behavior

The lab shows GNSS acquisition, sleep, battery use, queues, legal-airtime deferrals, packet airtime, link margin, collision/capture, relay scheduling and suppression, gateway decode, MQTT publication, archive commit and the reverse ACK path. Select a device after traffic to inspect the latest link budget term by term.

A fixed seed makes the same scenario reproducible. Humidity primarily changes wet-foliage loss; direct atmospheric attenuation at 868 MHz remains deliberately tiny. The built-in MQTT service is the one service needed by this product, including outages and durable archive receipts, not a general TCP broker.

Built and tested on every change

CI compiles the portable firmware core to WASM, compares golden policy vectors, runs deterministic engine scenarios and opens the real application in headless Chromium. It also continues to compile all five PlatformIO firmware targets and run the brokerless Python/SQLite service simulator.

cd components/simulator-web
npm ci
npm run build
npx playwright install --with-deps chromium
npm run test:browser

What the map cannot prove

It is an engineering model, not a calibrated digital twin. Antenna patterns, animal and terrain attenuation, real multipath/interference, GNSS satellites, ESP32 current, flash/brownout behavior, real TLS brokers and installed ERP still require hardware, infrastructure and field tests.

Important: never turn a displayed range or battery estimate into a product claim without measurements from the intended hardware, antenna, enclosure, mounting and site.
