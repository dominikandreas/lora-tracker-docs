Protocols

Contracts between replaceable components

Each boundary is versioned independently so firmware, services, and browser clients can be upgraded in a controlled order.

Identity

A canonical device ID is hashed into a compact public 64-bit routing identifier. A point remains stable across gateways:

<device_hash>:<boot_id>:<sequence>

The routing hash is not a credential, authentication mechanism, or encryption key.

LoRa frames

A packed little-endian envelope identifies transport version, message type, schema, flags, and device hash. History batches combine an absolute coordinate with compact signed deltas. Timestamp-aware batches store one absolute UTC value and unsigned variable-length deltas for later points.

MQTT event model

Point events are non-retained. Current state, availability, and status are retained. The gateway owns reception metadata such as RSSI and gateway identity; the tracker owns route and battery observations.

<base>/v1/trackers/<hash>/events/point
<base>/v1/trackers/<hash>/state
<base>/v1/trackers/<hash>/history/request
<base>/v1/trackers/<hash>/history/response/<request>

Configuration and onboarding

Tracker and gateway settings use CRC-protected, revisioned binary blobs with active and backup copies in NVS. HTTP and BLE onboarding apply the same transactional patch model, keeping transport-specific code away from validation logic.
