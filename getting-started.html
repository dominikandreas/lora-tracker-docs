Get started

Build the complete stack

The repository contains two PlatformIO firmware projects, a Python archiver, a static browser app, and a simulator that connects their contracts.

Requirements

  • PlatformIO Core or the VS Code PlatformIO extension
  • Python 3.11 or newer
  • Node.js for browser codec tests
  • An MQTT broker with TCP and WebSocket listeners for deployment

Compile a tracker

Copy the example secrets header, choose the environment matching the physical board, and compile:

cp components/tracker-firmware/secrets.example.h \
   components/tracker-firmware/secrets.h

pio run -d components/tracker-firmware \
  -e heltec_wifi_lora_32_v2

# ESP32-S3 Wireless Tracker alternative
pio run -d components/tracker-firmware \
  -e heltec_wireless_tracker
Factory seeds only. Values in secrets.h initialize an unprovisioned device. Active configuration is stored transactionally in NVS.

Compile the gateway

cp components/gateway-firmware/secrets.example.h \
   components/gateway-firmware/secrets.h

pio run -d components/gateway-firmware \
  -e heltec_wifi_lora_32_v2

Run the archiver and web app

cd components/archiver
python -m venv .venv
. .venv/bin/activate
pip install .
python -m equine_archiver

# In another terminal
cd components/web-app
python3 -m http.server 8080

Safe deployment order

  1. Back up the gateway configuration and archive database.
  2. Deploy the archiver.
  3. Flash and validate the gateway.
  4. Flash trackers and confirm acknowledgements.
  5. Deploy the web app and request recent history.
