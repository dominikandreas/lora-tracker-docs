Flash firmware in your browser

Choose the exact board and role, connect one device over USB, and install the latest release without PlatformIO.

Controlled field trials only. Release images are reproducible and radio traffic is authenticated, but Secure Boot, flash encryption and power-loss-safe tracker queueing remain production blockers.

1. Choose the exact hardware and role

The V2 tracker and V2 gateway share the same ESP32 chip. Automatic chip detection cannot tell their roles apart. Selecting the wrong role installs the wrong pin map and application.

2. Connect and install

SelectedWireless TrackerLoading the latest GitHub release…
Use a current desktop Chrome or Edge browser with Web Serial.Web Serial requires HTTPS and permission to access a USB device.
  1. Disconnect batteries and attach one board with a data-capable USB cable.
  2. Close PlatformIO and every serial monitor, then click Connect and install.
  3. For a new device, recovery, or role change, choose erase. Erase deletes configuration, keys, history and Wi-Fi credentials.
  4. Do not unplug until verification and reset finish. If entry fails, hold BOOT, tap RESET and retry.

3. Record first-boot credentials

Open the flasher’s serial console at 115200. An erased device generates a unique 20-character admin credential; record the admin / … line in your protected inventory. The generic release contains no shared password. Join the displayed tracker, gateway or lora-repeater-<suffix> AP, authenticate as admin, and complete onboarding. A gateway also needs the broker’s PEM root CA; a repeater needs the exact shared radio profile but no tracker key.

4. Verify what you installed

GitHub releases include SHA-256 checksums, ELF debugging files and signed build-provenance attestations. They identify the source of an artifact; they do not replace the still-missing device-enforced signed boot chain.

gh attestation verify lora-tracker-*.factory.bin \
  --repo dominikandreas/lora-tracker
