Keyless · bounded · bidirectional

Extend the radio path without exposing tracker keys

A repeater forwards encrypted tracker history toward a receiver and authenticated ACKs back toward the tracker. It never decrypts location data.

Field-trial feature. The software bounds loops, duplicates, memory and airtime. Installed RF behavior, regional compliance and worst-case multi-hop timing still require physical qualification.

One envelope, two directions

The six-byte mutable link header carries hop count and hop limit. The AES-256-GCM frame beneath it remains byte-for-byte unchanged, so only the tracker and registered receiver can authenticate or decrypt it.

Smart enough to avoid a flood

Nearby repeaters derive different deterministic priority slots. When one forwards first, peers that hear it cancel redundant pending copies. A farther node that only hears the new hop can continue the chain.

Build and onboard

pio run -d components/repeater-firmware \
  -e heltec_wifi_lora_32_v2

pio run -d components/repeater-firmware \
  -e heltec_wireless_tracker

On first boot, serial output at 115200 baud shows the protected setup AP and generated admin credential. Match every radio parameter to the tracker and gateway, choose the smallest useful hop cap, save and reboot. Repeating remains off until configuration validates.

Place for link margin, not convenience

POWER

Always-on infrastructure

Use protected mains/UPS or a measured solar design. A small tracker battery is unsuitable for continuous receive.

ANTENNA

High, clear, region-correct

Keep the vertical antenna clear of metal, wet foliage, bodies and animals; calculate installed ERP/EIRP.

TOPOLOGY

Measure both sides

Hidden repeaters can still collide. Test installed enclosures and remove nodes one at a time to document resilience.

Know what software cannot promise

The default 15-second tracker ACK window targets two hops at SF10/125 kHz. Higher spreading factors, four-hop paths and receiver processing can need more. The simulator validates link contracts and policy, but not propagation, capture effect, jamming, legal operation or real-time collision behavior.
